## 二.漏洞概述
目标站点全局引入存在高危安全缺陷的 `jQuery` 组件，受 `CVE-2020-11022/11023` 漏洞影响。攻击者可通过构造特殊 HTML 载荷，绕过前端防护并执行任意 JavaScript 代码，实现页面篡改、用户会话劫持、钓鱼攻击等操作，严重威胁站点安全与用户数据安全。
目前虽未发现公开可利用的可控输入渲染链路，但该高危组件长期未升级，后续若新增用户输入、内容回显等业务功能，可直接形成 DOM 注入 / XSS 攻击链，存在页面伪造、脚本劫持等潜在安全风险。

## 三.影响范围
- 新疆工程学院官网全站页面（全局加载`jQuery`）
- 所有访问该站点的用户浏览器会话

## 四.漏洞复现步骤
### 步骤一：漏洞触发验证（静默无害）
1. 在控制台执行以下代码，验证恶意代码可被解析执行：`var test = jQuery("<div><img src=x onerror='console.log(\"XSS Vulnerability Confirmed!\")'></div>");`
2. 结果：控制台输出 `XSS Vulnerability Confirmed!`，证明漏洞可被成功触发。
![](assets/jQuery%20DOM%20型%20XSS%20漏洞复现报告/file-20260420164708612.png)
### 步骤二：危害演示验证（非破坏性）
1. 在控制台执行以下代码，演示页面内容篡改风险：`jQuery("title").html("⚠️ 站点存在安全漏洞");`
2. 1. 结果：浏览器标签页标题被成功修改，证明攻击者可完全控制页面内容。

## 五.风险评估
1. **直接危害**：攻击者可执行任意 JavaScript 代码，实现 Cookie 窃取、会话劫持、页面篡改、钓鱼攻击等操作。
2. **潜在风险**：即使当前无直接可利用的用户输入点，该组件漏洞也为未来业务迭代埋下重大隐患 —— 只要新增任何用户可控的动态渲染点（如留言板、评论区、搜索结果回显），攻击者即可利用该漏洞触发反射型 / 存储型 XSS。
3. **合规影响**：该漏洞属于国家网络安全等级保护 2.0 标准中明确要求修复的高危第三方组件漏洞，未修复将影响站点安全合规性。

## （七）修复建议

### 1. 根治方案（推荐）

将站点的 `jQuery` 升级至安全版本，推荐：

- 兼容旧业务的过渡版本：`jQuery 1.12.4 → jQuery 1.12.9`（修复该漏洞的最后一个 1.x 版本）
- 长期稳定安全版本：`jQuery 3.7.1`（最新稳定版，彻底修复 DOM-XSS 缺陷）

### 2. 临时防护措施（升级前）

- 为站点添加 `Content-Security-Policy (CSP)` 响应头，限制未授权脚本执行。
- 对所有用户可控输入和动态输出内容，强制进行 HTML 实体转义，避免直接使用 `html()` 渲染用户输入内容。
- 禁用站点中不必要的 jQuery 动态渲染功能，优先使用 `text()` 纯文本渲染。
