关系与非关系区别：

1.存储方式不同：关系数据库为表格形式，非关系数据库为文档或图结构

2.拓展方式不同：关系数据库可纵向扩展，可提高处理能力，非关系数据库为天然分布式，通过更多的数据服务器来分担负载

3.事务处理支持不同：关系数据库善于处理事务原子性细粒度控制，方便事务回滚，非关系数据库着重处理大数据

## 3.MySQL数据库增删改查
> 库中有表，表中有数据
>

> 先指定数据库，才能新增数据表
>

show databases;	查看所有数据库

use name(数据库的命字); 进入指定数据库

show tables;		查看当前数据库中所有的表

desc name(表名);	查看指定表的结构



create database name(库名);	创建一个数据库，库名为指定内容

drop database name(库名);	删除指定数据库

create table name(表明) (字段名 字段类型 [字段约束]，字段名2 字段类型[字段约束],......);	创建一个表，表明为指定内容，创建在所在库下，字段名为列名（尽量避开中文或按照规则加引号），字段类型表示数据类型，[]内可省略，字段约束表明性质，对列的限制，如：主键、非空、唯一、自增、默认值

drop table name（表名）;	删除指定表

alter table name(表名) add name(字段名) int(字段类型);向指定表中添加特定字段

alter table name(表名) drop name(字段名);删除指定表中的指定字段

alter table name(表名) change id age int;将指定表中id字段名改为int类型的age字段名	数据会尽可能地保留





字符串：varchar(长度)	该数据类型必须指定长度，int类型可以不指定长度





insert into day values (数值,数值,....),(....);将数值插入day表，每个逗号隔开一列（换一列），逗号后的括号是下一行的数据，varchar数据类型插入时要用引号包裹

delete from day where id=4 ;删除day表id=4的数据

update day set id=6 where id=8;将day表id=8的数据改为id=6 set后面跟修改内容 where后面跟判断条件 为防止误判尽量使用唯一怕不断条件

update day set id=6,name=sb where id=5;对符合条件的数据进行批量修改，修改内容用,隔开



select *from day where id=5;查询day表 id=5的数据 *表明返回所有列的数据

select id,username from day where id-5;查询id=5的数据的id,username列数据

select username from day;查询day表username列的所有数据



select *from day order by id/1 (asc);基于id列或字段1列将day表的数据进行升序排列查询 （）可省略默认asc升序 desc降序 列数超过现实时会报错

select *from day limit 0,2;查询day表从第1行开始向后取2行的数据 0类似数组下标序号 超出索引不会报错

select group_concat(id,username) from day;将day表id,username两列的数据全部取出 一行输出

select group_concat(username) from demo01.admin;跨库查询，查询demo01库的admin表username数据 一行输出	增删查改均可通过这种方式跨库查询from 库名.表名

select *from day where username like 'd%/%d';模糊查询 查询day表username字段中以d开头/结尾的数据并返回全部字段 x%g x开头g结尾 %z% 字段中包含z的

## 4.高阶玩法
### 逻辑运算符
and	需要满足同时两个或更多个条件	 并且	where id=5 and age=10	需要同时满足两个条件

or	只需要满足多个条件中的一个条件即可	where id=5 or age=10 需要满足任意一个的条件

not	取反								where id not in (1) id不等于1的符合条件

万能密码 or 1=1   SQL注入

### 联合查询
union

```sql
select * from gay union select * from day;
#联合查询gay表和day表中的所有数据	表1表2中的相同数据只显示一次
select * from gay union select * from 1，2，3，4;#表1有4个字段
select * from gay union select * from day where id=1 and age=20;
#联合查询gay表所有数据和day表符合条件的数据
select * from gay union select * from day where id=1 and 1=2;
#表2的条件不成立，只显示表1数据
```

特性：

+ 两张表的字段数相同，不然联合查询会失败
+ 多张表里面重复的数据，只会输出1次

## 5.information_schema数据库的运用
> information_schema是MySQL的内置数据库（5.0版本后自带），存放着MySQL服务器所维护的所有其他数据库的信息。
>

information_schema数据库表说明：

+ SCHEMAT表
    - 提供当前MySQL示例中的所有数据库的信息。
    - show databases的结果取之此表
+ TABLES表
    - 提供关于数据库中表的信息（包括视图） 详细表达了某个某个表属于哪个schema、表类型、表引擎、创建时间等信息。
    - show tables from schemaname的结果取之此表
+ COLUMNS表
    - 提供表中的列信息（字段名） 详细表述了某张表的所有列以及每个列的信息
    - show columns from schemaname.tablename的结果取之此表

熟练使用，获取目标服务器数据库——库、表、字段名
