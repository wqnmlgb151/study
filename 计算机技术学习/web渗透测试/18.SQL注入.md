## 1.sql注入原理
> sql注入又被称为漏洞之王
>

+ 注入攻击漏洞往往是应用程序缺少对输入进行安全性检查引用
+ 攻击者可以把一些包含攻击代码当命令或查询语句送给解释器，从而执行计划外的命令或者未授权访问数据
+ 通常能SQL查询、LDAP查询、OS命令、程序参数等中出现
+ 用户输入的数据（传递的参数）被拼接到了代码当中，当作了SQL语句进行执行

---

## 2.sql注入常用函数
+ 重要函数：
    - `version()`		数据库版本
    - `database()`		当前数据库名
    - `user()`			用户名
    - `current_user()`	当前用户名
    - `system_user()`	系统用户名
    - `@@datadir`		数据库路径
    - `@@version_compile_os`操作系统版
+ 重要数据库：
    - `information_schema`	包含MySQL中的所有数据库的信息
+ 重要的表：
    - `schemata`	mysql中的所有数据库信息
    - `schema_name`所有数据库名
    - `tables`		数据库表中信息
    - `table_schema`记录数据库名
    - `table_name`	记录数据库名
    - `columns`	列信息
    - `column_name`列字段名

### 总结
判断闭合的方法：

在加入猜测的符号后再加上`and 1=1`和`and 1=2`来进行判断

如果两者界面一样，猜对了，闭合完成

如果界面不一样，猜错了，闭合失败

在谷歌、fofa中可以查询关键词来找含有sql漏洞的

### 判断字段数
> `order by` 排序命令

利用`order by`后面跟的数字超出实际列数会报错来判断字段数

### 判断显错位
使用`union`
在语句后面加上不可能的条件再拼接联合查询：`?id=-1 union select 1,2,3--+`
使得在前端显示的是我们联合查询要的结果，判断显错位来在对应位置放置对应函数获取对应信息：`?id=-1 union select 1,database(),3__+`(使用的靶场，上一步已确认2 3会显示，所以在2的位置放置函数)
### 判断表名、列名、具体数据
> mysql有内置数据库information_schema存放着渗透所需的大部分信息，且访问权限极低，是个人都能看

|表名|存储内容|渗透中的核心用途|
|---|---|---|
|`SCHEMATA`|所有数据库的名称|第一步：获取服务器上的**所有数据库名**（比如找到 `admin`/`user`/`mysql` 等核心库）|
|`TABLES`|所有表的信息（库名 + 表名 + 引擎等）|第二步：针对目标库，获取**所有表名**（比如找到 `user`/`admin`/`account` 等存账号的表）|
|`COLUMNS`|所有字段的信息（表名 + 字段名 + 类型）|第三步：针对目标表，获取**所有字段名**（比如找到 `username`/`password`/`id` 等核心字段）|
|`USER_PRIVILEGES`|数据库用户的权限信息|进阶：判断当前用户权限（比如是否能读写文件、执行系统命令）|

- `select table_name from information_schema.tables where table_schema='xiaoming'`获取`xiaoming`库的表名
	- 可以运用`limit`来获取不同的表名（部分有限制，直接使用上述命令只能获取第一个表名）
	- 可以运用`group_concat()`来获取所有，将字段名放到`()`内（table_name）
- `select column_name from information_schema.columns where table_name='gay'`获取`gay`表的字段名
